[ 동작 요약 ]
  ▸ 한 라운드 = 활성화된 모든 사이트에 같은 글 + 같은 이미지 동시 게시
  ▸ 라운드 간격은 공통 (예: 7200초 = 2시간마다)
  ▸ 아이보스는 일일 2회 도달 시 자동 건너뜀 (자정에 카운터 리셋)
  ▸ messages/ 의 .txt 들이 라운드마다 순차/랜덤으로 선택됨

[ 시작 전 준비 ]
  ▸ SellClubBot.exe 옆에 두 폴더가 자동 생성됩니다.
      messages\   : .txt 파일을 여러 개 넣어둡니다.
                    첫 줄 = 제목, 그 아래부터 = 본문
      images\     : 첨부할 이미지 (.jpg/.png/.gif 등, 1MB 이하)
  ▸ 셀클럽/마멘토: multipart 파일첨부 방식
  ▸ 아이보스: base64 data URL 본문 임베드 방식 (브라우저 복사-붙여넣기와 동일)
    → 1MB 초과 이미지는 본문 길이 초과 우려로 자동 제외

[ 사용 절차 ]
  1) exe 실행 → 사용자 인증창 (구글시트 등록 ID/PW)
  2) 메인창 탭 메뉴
       [셀클럽]   : 로그인 + 카테고리/등록분류/거래상황/전화·핸드폰/게시물형태
       [마멘토]   : 로그인 + 자유홍보광고 하위 게시판 선택 + 카테고리(ca_name)
       [아이보스] : 로그인 + 구분(매체/유형) + 회사명/담당자명/연락처
       [공통]     : 메시지/이미지 폴더 새로고침, 로테이션 모드 선택
  3) 하단에서 반복횟수 / 간격(초) / ±랜덤(초) 설정
  4) [발송 시작] 클릭

[ 일일 한도 ]
  ▸ 셀클럽 / 마멘토: 무제한
  ▸ 아이보스: 일일 2회 (daily_state.json 에 저장, 자정 자동 리셋)

[ 포인트 차감 (셀클럽) ]
  ▸ 게시물형태(wr_1) 옵션마다 포인트가 차감됩니다.
      3=기본 -1500P / 2=굵게 -2000P / 1=급등 -2500P / 0=추천 -151500P
  ▸ 잔액 부족 시 등록 실패. 포인트 미리 확인하세요.

[ 검증 / 문제 발생 시 ]
  ▸ 단독 로그인 테스트:
      cd "프로그램 폴더"
      python test_login.py <셀클럽ID> <셀클럽PW>
  ▸ 3사이트 통합 E2E (실제 게시는 안 함):
      $env:SC_ID="..." ; $env:SC_PW="..."
      $env:MM_ID="..." ; $env:MM_PW="..."
      $env:IB_ID="..." ; $env:IB_PW="..."
      python test_e2e.py
  ▸ "로그인 실패" → 셀클럽/마멘토/아이보스 ID/PW 다시 확인
  ▸ "1MB 제한 초과" → images/ 의 이미지 용량 축소
  ▸ "등록되지 않은 사용자" → 관리자에게 등록 요청

[ 파일 구조 ]
  main.py         — GUI 진입점
  auth.py         — 구글시트 사용자 인증
  base.py         — 공통 BoardClient / WriteResult
  sellclub.py     — 셀클럽 클라이언트 (EUC-KR, 그누보드4)
  mamentor.py     — 마멘토 클라이언트 (UTF-8, 그누보드5, 30+ 하위게시판)
  iboss.py        — 아이보스 클라이언트 (커스텀 CMS, Summernote 본문, 일일 2회)
  multibot.py     — 다중사이트 오케스트레이터 (일일 카운터 자동 관리)
  scheduler.py    — 반복 발송 스케줄러 (별도 스레드)
  content.py      — 메시지/이미지 로테이션
  config.py       — 상수
  paths.py        — exe 배포 시 경로 처리
  daily_state.json — 사이트별 일일 게시 카운터 (자동 생성)
  settings.json   — 저장된 사용자 설정 (자동 생성)
